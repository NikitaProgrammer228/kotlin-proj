# Решение ошибки "Can't push refs to remote"

## Проблема
При попытке синхронизации (push) появляется ошибка:
> "Can't push refs to remote. Try running 'Pull' first to integrate your changes."

## Причина
В удалённом репозитории есть изменения (коммиты), которых нет в вашей локальной копии. Git не позволяет делать push, пока вы не получите эти изменения.

## Решение

### Вариант 1: Pull + Push (рекомендуется)

**Шаг 1:** Получить изменения с удалённого репозитория
```powershell
git pull
```

Если есть конфликты, Git попросит их разрешить. После разрешения:
```powershell
git add .
git commit -m "Merge remote changes"
```

**Шаг 2:** Отправить ваши изменения
```powershell
git push
```

### Вариант 2: Pull с rebase (если хотите линейную историю)

```powershell
git pull --rebase
```

Если есть конфликты:
1. Разрешите конфликты в файлах
2. `git add .`
3. `git rebase --continue`

Затем:
```powershell
git push
```

### Вариант 3: Если вы уверены, что ваши изменения важнее

⚠️ **ВНИМАНИЕ:** Это перезапишет удалённые изменения!

```powershell
git push --force
```

**Используйте только если:**
- Вы работаете один
- Вы уверены, что удалённые изменения не нужны
- Вы понимаете последствия

## Если у вас есть незакоммиченные изменения

**Сначала сохраните ваши изменения:**

```powershell
# Посмотреть, что изменено
git status

# Добавить все изменения
git add .

# Закоммитить
git commit -m "Your commit message"

# Затем pull
git pull

# И push
git push
```

## Если есть конфликты при pull

1. Git покажет файлы с конфликтами
2. Откройте эти файлы в редакторе
3. Найдите маркеры конфликтов:
   ```
   <<<<<<< HEAD
   ваш код
   =======
   код из удалённого репозитория
   >>>>>>> branch-name
   ```
4. Выберите нужный код или объедините оба варианта
5. Удалите маркеры конфликтов
6. Сохраните файлы
7. Выполните:
   ```powershell
   git add .
   git commit -m "Resolve merge conflicts"
   git push
   ```

## Проверка статуса

Перед push всегда проверяйте:
```powershell
# Посмотреть статус
git status

# Посмотреть, что изменилось
git log --oneline --graph --all -10
```

## Рекомендуемый порядок действий

1. ✅ `git status` - проверить текущее состояние
2. ✅ `git add .` - добавить изменения (если есть незакоммиченные)
3. ✅ `git commit -m "..."` - закоммитить (если есть изменения)
4. ✅ `git pull` - получить изменения с удалённого репозитория
5. ✅ Разрешить конфликты (если есть)
6. ✅ `git push` - отправить изменения
